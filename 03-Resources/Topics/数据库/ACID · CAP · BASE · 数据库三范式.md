# ACID · CAP · BASE · 数据库三范式

## Summary

这四组概念经常被混着背，但它们其实在回答四类不同问题：事务正确性、分布式约束、一致性策略和表结构设计。

## 一张表先分清

| 概念 | 它回答的问题 |
|------|--------------|
| 三范式 | 表结构怎么设计，减少冗余和更新异常 |
| ACID | 单机事务正确性怎么保证 |
| CAP | 分布式系统在网络分区下有哪些根本取舍 |
| BASE | 分布式系统如何以最终一致换取更高可用性 |

## 核心判断

- `ACID` 更像关系型事务的底线。
- `CAP` 不是“随便选两个”，而是在分区出现时讨论取舍。
- `BASE` 是对 AP 路线的工程化接受，不是“不一致也无所谓”。
- 三范式属于数据建模问题，不该和事务语义混成一堆。
