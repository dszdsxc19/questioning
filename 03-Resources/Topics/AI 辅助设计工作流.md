# AI 辅助设计工作流

## Summary

AI 辅助设计的关键不是让模型“自由发挥”，而是先建立风格约束、组件基座和评审标准，再让 AI 放大执行效率。

## 这页解决什么问题

这页用来沉淀一个稳定方法：当 AI 开始参与前端和界面设计时，人应该约束什么，AI 应该执行什么。

## 选用原则

收录到这页的内容，优先满足以下条件：

- 能帮助你减少 UI 随机性和审美漂移
- 能沉淀成长期可复用的工作流，而不是某次项目经验
- 能连接到组件库、prompt 结构和评审方式

## 核心内容

### 设计角色的变化

AI 时代的关键变化不是“设计消失了”，而是人从手工产出者转为规则制定者和审美把关者。

### 一个更稳定的工作流

1. 先定义设计目标和审美方向
2. 选择 AI 易于理解和修改的组件基座
3. 建立主题变量、排版和视觉约束
4. 用 prompt 或 skill 驱动 AI 产出
5. 由人做最终审美判断和取舍

### 为什么先做约束系统

没有约束时，AI 往往会产出：

- 风格飘忽
- 组件不一致
- 动效堆砌
- 代码难以长期维护

约束系统的作用，是把 AI 的速度放进可控边界里。

### 三个高杠杆抓手

- 组件基座：优先选 AI 读得懂、改得稳的组件库
- 主题系统：把颜色、排版、间距和语义色变量化
- Prompt 结构：让需求描述从“好看一点”升级为可执行规范

### 与 prompt 的关系

Prompt 在这里不是素材库，而是把设计词汇和场景约束转成稳定执行接口的方式。
