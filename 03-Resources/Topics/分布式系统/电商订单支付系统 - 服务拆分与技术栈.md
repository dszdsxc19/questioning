# 电商订单支付系统 - 服务拆分与技术栈

## Summary

这页从一个电商订单与支付系统出发，解释为什么要拆分微服务，以及每个服务的职责、数据存储和事件流如何协作。

## 服务拆分

- API Gateway：统一入口、鉴权、限流、熔断、日志追踪
- 用户服务：注册登录、用户信息、地址管理
- 商品服务：商品 CRUD、搜索、详情页
- 订单服务：创建订单、订单状态机、查询
- 库存服务：库存查询、锁定、释放、扣减
- 优惠券服务：发券、领券、核销、有效期管理
- 支付服务：发起支付、回调、退款
- 账务服务：流水记账、对账、财务报表
- 通知服务：短信、邮件、Push、站内信

## 事件驱动

同步调用的问题是：耦合强、容易超时、无法削峰。

事件驱动的解法是：

```text
订单服务 -> Publish -> Kafka: order.created
                         -> 库存服务
                         -> 优惠券服务
                         -> 通知服务
```
