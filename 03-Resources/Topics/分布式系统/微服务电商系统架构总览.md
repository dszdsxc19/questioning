# 微服务电商系统架构总览

## Summary

这页是分布式系统主题的总览页，用电商订单与支付系统作为场景，把服务拆分、事件驱动、一致性、链路追踪和网关这些核心问题串起来。

## 为什么要拆分微服务

单体应用的痛点：

- 发布耦合
- 扩容粒度粗
- 技术栈锁定
- 故障蔓延

微服务的核心思想：按业务边界拆分，每个服务独立部署、独立扩容、独立演化。

## 服务拆分：按领域边界划分

核心服务一览：

- API Gateway
- 用户服务
- 商品服务
- 订单服务
- 支付服务
- 库存服务
- 优惠券服务
- 账务服务
- 通知服务

## 事件驱动：消息队列如何串联服务

完整下单事件流：

1. 用户提交订单，订单服务创建待支付订单并发布 `order.created`
2. 库存服务消费事件，锁定库存
3. 优惠券服务消费事件，核销优惠券
4. 支付服务发起支付，等待异步回调
5. 支付成功后发布 `payment.success`
6. 订单服务更新订单状态
7. 库存服务正式扣减库存
8. 账务服务记录流水
9. 通知服务发送成功通知

## 技术栈全景图

从客户端层、API Gateway 层、业务服务层、消息队列层到底层观测与部署设施，这页承担的是总览导航，不是每一层的细节文档。
