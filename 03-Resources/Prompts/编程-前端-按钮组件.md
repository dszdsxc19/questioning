# 🔘 前端提示词 · 按钮组件

> 好的提示词不只是让 AI 能执行，更是在训练自己的**设计词汇**。
> 每读一条提示词，都问自己：这个词描述的是什么视觉效果？我见过吗？我以前会怎么描述它？

---

## 提示词集合

### 一级按钮 · 强行动引导（Primary CTA）

> 页面最核心的行动指令，视觉权重最高，通常只有一个。

| | 提示词 |
|---|---|
| **中文** | 交互式行动按钮，带有右箭头 SVG 图标，并通过基于**光标位置**的 CSS 变量实现**自定义光晕**效果。适用于**首页视觉区**或**引导流程**提示。 |
| **English** | Interactive call-to-action button featuring a right-arrow SVG icon and custom **glow effect** driven by CSS variables for **cursor position**. Ideal for **hero sections** or **onboarding prompts**. |

**解读**：光晕跟着鼠标走（mouse-tracking glow），通过 JS 把鼠标坐标写入 CSS 变量，再用 `radial-gradient` 实现。这是目前很流行的"有温度"的交互。

---

### 次级按钮 · 高互动引导（Secondary CTA）

> 辅助行动，优先级低于一级，通常在表单末尾或配合一级按钮使用。

| | 提示词 |
|---|---|
| **中文** | 使用 Tailwind CSS，包含**滑入式模糊入场动画**、随鼠标移动的**动态光晕**、悬停时的**下划线延展**，以及**箭头图标的位移效果**。 |
| **English** | Call-to-action button styled with Tailwind CSS featuring **slide-in blur entrance**, **dynamic mouse-move glow**, **underline expansion** and **arrow icon shift** on hover. Ideal for landing pages or forms requiring an engaging "Book a Call" prompt. |

**解读**：四个独立效果叠加——入场动画 + 悬停光晕 + 文字下划线 + 图标位移。每一个单独都不复杂，叠加后就有强烈的"精致感"。

---

### 文字按钮 · 未来感风格（Ghost / Text CTA）

> 无背景，靠光效和动画撑起视觉，适合暗色背景或科幻风界面。

| | 提示词 |
|---|---|
| **中文** | 带有**霓虹紫色发光效果**的动画按钮，悬停时会以**打字光标动画**呈现**彩色文字**。适用于**现代感、未来感界面**。 |
| **English** | An animated button with **neon purple glow** effect that reveals colored text with **typing cursor animation** on hover. Perfect for **modern, futuristic interfaces**. |

**解读**：霓虹发光 = `box-shadow` 多层叠加，通常配合高饱和色（紫、蓝、青）。打字动画 = `steps()` + `overflow: hidden` + `border-right` 模拟光标。

---

## 📖 词库逆向

> 从上面三条提示词里提炼的设计词汇，这是你下次描述 UI 时的弹药库。

### 动效词汇

| 英文术语 | 中文 | 视觉效果描述 |
|---|---|---|
| Slide-in blur entrance | 滑入式模糊入场动画 | 元素从模糊状态淡入并上移，有"从虚空召唤"的感觉 |
| Typing cursor animation | 打字光标动画 | 文字逐字出现，末尾有闪烁竖线，模拟真人打字 |
| Underline expansion | 下划线延展 | 鼠标悬停时，下划线从左向右延伸覆盖整个文字 |
| Arrow icon shift | 箭头图标位移 | 悬停时箭头向右位移几像素，强化"向前走"的隐喻 |
| Mouse-tracking glow | 追踪鼠标的光晕 | 光晕的中心跟随鼠标位置移动，让按钮"感知"用户 |

### 光效词汇

| 英文术语 | 中文 | 视觉效果描述 |
|---|---|---|
| Neon glow | 霓虹发光 | 元素边缘有强烈的彩色光晕，像霓虹灯管 |
| Custom glow | 自定义光晕 | 通用描述，通常指用 CSS 变量控制强度/颜色 |
| Dynamic mouse-move glow | 动态鼠标光晕 | 光晕随鼠标移动而变化，有响应感 |
| Radial gradient spotlight | 径向渐变聚光灯 | 以某点为圆心向外渐变，模拟灯光照射 |

### 适用场景词汇

| 英文 | 中文 | 说明 |
|---|---|---|
| Hero section | 首页视觉区 | 页面最顶部的大图区域，第一眼看到的 |
| Onboarding prompts | 引导流程 | 新用户注册/入门的引导步骤 |
| Landing page | 落地页 | 广告/营销导向的单页，以转化为目标 |
| Above the fold | 首屏 | 不用滚动就能看到的区域 |
| Call to action (CTA) | 行动号召 | 驱动用户点击/注册/购买的按钮或链接 |

## 🧩 提示词结构解析

分析这三条提示词，好的 UI 提示词都由五个部分构成：

```
[组件类型] + [视觉风格] + [核心特效] + [交互行为] + [适用场景]
```

| 部分 | 示例 | 你以前可能忽略的 |
|---|---|---|
| **组件类型** | 行动按钮、文字按钮 | — |
| **视觉风格** | 霓虹、玻璃拟态、赛博朋克 | — |
| **核心特效** | 光晕、发光、模糊入场 | 要有具体特效名，不能只说"好看" |
| **交互行为** | 悬停时位移、悬停时下划线 | 要区分默认态/悬停态/点击态 |
| **适用场景** | hero 区、落地页、表单末尾 | ⚠️ 最常被忽略，但影响 AI 理解意图 |

## ✍️ 提示词自写模板

下次写新组件提示词时，套用此模板：

```
一个[组件类型]，风格为[视觉风格]。
包含以下效果：[特效1]、[特效2]、[特效N]。
交互行为：默认状态[描述]，悬停时[描述]，点击时[描述]。
适用场景：[场景描述]。
技术栈：Tailwind CSS + shadcn/ui。
```

**填写示例**（自己练习）：
```
一个卡片组件，风格为玻璃拟态（glassmorphism）。
包含以下效果：背景模糊、半透明边框、内发光。
交互行为：默认状态有微弱阴影，悬停时边框高亮并轻微上浮。
适用场景：功能介绍区（features section）或定价页。
技术栈：Tailwind CSS + shadcn/ui。
```
