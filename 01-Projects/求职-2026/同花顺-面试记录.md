## 算法题

### 题目：二维数组查找目标值

**题目描述**：
在一个 `n × m` 二维数组中，每一行都从左到右递增，每一列都从上到下递增。给定一个函数，判断二维数组中是否存在某整数。

**示例**：
```
矩阵：
[
  [1,  4,  7, 11, 15],
  [2,  5,  8, 12, 19],
  [3,  6,  9, 16, 22],
  [10, 13, 14, 17, 24],
  [18, 21, 23, 26, 30]
]

目标值: 5 → true
目标值: 20 → false
```

**我的回答**：{{待补充}}

**参考解法（JavaScript）**：

```javascript
/**
 * 在有序二维数组中查找目标值
 * 时间复杂度: O(n + m)
 * 空间复杂度: O(1)
 *
 * 思路：从右上角开始查找
 * - 如果当前值 > target，向左移动（排除当前列）
 * - 如果当前值 < target，向下移动（排除当前行）
 * - 如果当前值 === target，返回 true
 */
function findTargetInMatrix(matrix, target) {
  if (!matrix || matrix.length === 0 || matrix[0].length === 0) {
    return false
  }

  const rows = matrix.length
  const cols = matrix[0].length

  // 从右上角开始
  let row = 0
  let col = cols - 1

  while (row < rows && col >= 0) {
    const current = matrix[row][col]

    if (current === target) {
      return true
    } else if (current > target) {
      // 当前值大于目标，向左移动（排除当前列）
      col--
    } else {
      // 当前值小于目标，向下移动（排除当前行）
      row++
    }
  }

  return false
}

// 测试
const matrix = [
  [1, 4, 7, 11, 15],
  [2, 5, 8, 12, 19],
  [3, 6, 9, 16, 22],
  [10, 13, 14, 17, 24],
  [18, 21, 23, 26, 30]
]

console.log(findTargetInMatrix(matrix, 5))   // true
console.log(findTargetInMatrix(matrix, 20))  // false
console.log(findTargetInMatrix(matrix, 30))  // true
```

**思路图解**：
```
从右上角 (row=0, col=4) 开始：
┌─────┬─────┬─────┬─────┬─────┐
│  1  │  4  │  7  │ 11  │ 15  │  ← 起点
├─────┼─────┼─────┼─────┼─────┤
│  2  │  5  │  8  │ 12  │ 19  │
├─────┼─────┼─────┼─────┼─────┤
│  3  │  6  │  9  │ 16  │ 22  │
├─────┼─────┼─────┼─────┼─────┤
│ 10  │ 13  │ 14  │ 17  │ 24  │
├─────┼─────┼─────┼─────┼─────┤
│ 18  │ 21  │ 23  │ 26  │ 30  │
└─────┴─────┴─────┴─────┴─────┘

查找 target = 5:
1. 当前值 15 > 5，向左移动 col--
2. 当前值 11 > 5，向左移动 col--
3. 当前值 7 > 5，向左移动 col--
4. 当前值 4 < 5，向下移动 row++
5. 当前值 5 === 5，找到！返回 true
```

### 1. 大数据量渲染优化

**问题**：一个表单有大量数据（很大的 JSON）传给前端，怎么样渲染能够不卡顿？

**我的回答**：虚拟列表 ✅（思路正确，需补充优劣势分析）

**补充要点**：

#### 虚拟列表（Virtual Scrolling）

**核心思想**：只渲染可视区域内的元素，通过计算滚动位置动态更新渲染内容。

**优势**：
- 🚀 **性能优秀**：无论数据量多大，DOM 节点数量恒定
- 💾 **内存占用低**：不维护大量 DOM 元素
- ⚡ **滚动流畅**：60fps 无压力
- 📈 **可扩展性强**：支持百万级数据

**劣势**：
- 📏 **高度固定**：需要知道或计算每个 item 的高度
- 🔄 **滚动复杂**：需要处理滚动事件和位置计算
- 🎨 **样式受限**：不能使用基于父元素的 CSS 选择器
- 📱 **兼容性**：移动端惯性滚动处理较复杂

**实现要点**：
```javascript
// 虚拟列表核心参数
const virtualList = {
  itemHeight: 50,        // 每项高度
  containerHeight: 600,  // 可视区域高度
  totalItems: 10000,     // 总数据量
  scrollTop: 0,          // 滚动距离

  // 计算可视区域
  get visibleRange() {
    const start = Math.floor(this.scrollTop / this.itemHeight)
    const end = Math.ceil((this.scrollTop + this.containerHeight) / this.itemHeight)
    return { start: Math.max(0, start), end: Math.min(this.totalItems, end) }
  }
}
```

**完整实现示例**：
```javascript
class VirtualList {
  constructor(options) {
    this.containerHeight = options.containerHeight || 600
    this.itemHeight = options.itemHeight || 50
    this.data = options.data || []
    this.container = options.container

    this.visibleCount = Math.ceil(this.containerHeight / this.itemHeight)
    this.scrollTop = 0

    // 创建容器结构
    this.init()
  }

  init() {
    // 总高度容器（撑开滚动条）
    this.phantom = document.createElement('div')
    this.phantom.style.height = `${this.data.length * this.itemHeight}px`

    // 可视内容容器
    this.content = document.createElement('div')
    this.content.style.position = 'relative'
    this.content.style.top = '0px'

    this.container.appendChild(this.phantom)
    this.container.appendChild(this.content)

    // 监听滚动
    this.container.addEventListener('scroll', () => this.onScroll())
    this.render()
  }

  onScroll() {
    this.scrollTop = this.container.scrollTop
    this.render()
  }

  get visibleRange() {
    const start = Math.floor(this.scrollTop / this.itemHeight)
    // 多渲染几项避免滚动时白屏
    const buffer = 3
    const end = start + this.visibleCount + buffer * 2
    return {
      start: Math.max(0, start - buffer),
      end: Math.min(this.data.length, end)
    }
  }

  render() {
    const { start, end } = this.visibleRange
    const fragment = document.createDocumentFragment()

    for (let i = start; i < end; i++) {
      const item = document.createElement('div')
      item.style.height = `${this.itemHeight}px`
      item.style.position = 'absolute'
      item.style.top = `${i * this.itemHeight}px`
      item.style.width = '100%'
      item.textContent = this.data[i]
      this.content.appendChild(item)
    }

    this.content.innerHTML = ''
    this.content.appendChild(fragment)
  }
}

// 使用
const vList = new VirtualList({
  container: document.getElementById('list-container'),
  itemHeight: 50,
  data: Array.from({ length: 10000 }, (_, i) => `Item ${i}`)
})
```

**核心原理图解**：
```
┌─────────────────────────────────────┐
│   可视区域 (viewport)                │  ← containerHeight
│   ┌─────────────────────────────┐   │
│   │ Item 5 (scrollTop: 250)     │   │  ← 实际渲染
│   │ Item 6                      │   │
│   │ Item 7                      │   │
│   │ Item 8                      │   │
│   │ Item 9                      │   │
│   └─────────────────────────────┘   │
├─────────────────────────────────────┤
│   总高度 (phantom)                   │  ← data.length * itemHeight
│   50000px (10000 items × 50px)      │     用于撑开滚动条
└─────────────────────────────────────┘

滚动时：
1. 监听 scroll 事件，获取 scrollTop
2. 计算 start = floor(scrollTop / itemHeight)
3. 只渲染 [start, start + visibleCount] 范围的元素
4. 用 transform: translateY() 或 absolute 定位
```

**动态高度处理**：
```javascript
// 如果 item 高度不固定，需要先测量位置
class DynamicVirtualList {

    this.positions = []  // 缓存每个 item 的位置信息
    this.estimatedItemHeight = 50  // 估算高度
  }

  updatePositions() {
    const nodes = this.content.children
    for (let i = 0; i < nodes.length; i++) {
      this.positions[this.startIndex + i] = {
        height: nodes[i].offsetHeight,
        top: i === 0 ? 0 : this.positions[this.startIndex + i - 1].top +

      }
    }

}
```

### 2. GET vs POST 请求

**问题**：POST 请求和 GET 请求有什么区别？

**我的回答**：{{待补充}}

**补充要点**：

| 维度 | GET | POST |
|------|-----|------|
| **数据位置** | URL 参数 | Request Body |
| **数据大小** | 受 URL 长度限制（通常 2KB-8KB） | 理论无限制 |
| **编码类型** | application/x-www-form-urlencoded | 多种（multipart/form-data、application/json 等） |
| **缓存** | 可被缓存 | 默认不缓存 |
| **书签/历史** | 可保存为书签，出现在历史记录 | 不会 |
| **安全性** | 参数明文显示在 URL | 相对更安全（但仍是明文传输） |
| **幂等性** | 幂等（多次请求结果相同） | 非幂等 |
| **语义** | 获取资源 | 提交数据/创建资源 |

**深入理解：幂等性**

#### 什么是幂等性？

> **幂等性（Idempotent）**：同一个操作执行多次，产生的结果与执行一次相同。

数学定义：`f(x) = f(f(x))`

#### 为什么 GET 是幂等的？

GET 的语义是"获取资源"，不改变服务器状态：

```
GET /api/user/123
```

无论请求多少次，服务器上的用户数据不会改变，返回的结果也是相同的（忽略数据时效性）。

**示例**：
```javascript
// 幂等：多次请求不会产生副作用
fetch('/api/articles/1')    // 获取文章
fetch('/api/articles/1')    // 再获取一次，文章不变
fetch('/api/articles/1')    // 再获取一次...
```

#### 为什么 POST 是非幂等的？

POST 的语义是"提交数据/创建资源"，每次请求都会在服务器上创建新资源：

```
POST /api/articles
{ title: "新文章", content: "..." }
```

每次请求都会创建一篇新文章，多次请求产生多条记录。

**示例**：
```javascript
// 非幂等：每次请求都创建新订单
fetch('/api/orders', {
  method: 'POST',
  body: JSON.stringify({ productId: 1, quantity: 2 })
})
// 第1次：创建订单 #1001
// 第2次：创建订单 #1002
// 第3次：创建订单 #1003
// ❌ 用户可能重复下单！
```

#### 幂等性的实际意义

**1. 自动重试安全**
```
GET 请求失败 → 直接重试 ✅（安全）
POST 请求失败 → 重试前需确认 ❌（可能重复创建）
```

**2. 缓存策略**
```
GET：可以缓存（幂等，缓存安全）
POST：不缓存（非幂等，缓存可能出错）
```

**3. 浏览器行为**
```
- 刷新 GET：浏览器直接重新请求
- 刷新 POST：浏览器提示"确认重新提交表单"
```

#### 如何让 POST 变幂等？

**方法1：使用幂等键（Idempotency Key）**
```javascript
// 客户端生成唯一 ID
const idempotencyKey = crypto.randomUUID()

fetch('/api/orders', {
  method: 'POST',
  headers: {
    'Idempotency-Key': idempotencyKey,  // 幂等键
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ productId: 1, quantity: 2 })
})

// 服务器逻辑：
// 1. 检查 Idempotency-Key 是否已处理
// 2. 如果已处理，返回缓存的结果
// 3. 如果未处理，正常处理并缓存结果
```

**方法2：使用 PUT 代替 POST（更新场景）**
```
POST /api/user/123  → 可能创建多个用户记录
PUT /api/user/123   → 幂等，多次请求结果相同
```

**方法3：业务层去重**
```javascript
// 前端：防抖 + loading 状态
const [loading, setLoading] = useState(false)

const handleSubmit = async () => {
  if (loading) return  // 防止重复点击
  setLoading(true)
  await createOrder(data)
  setLoading(false)
}
```

#### HTTP 方法幂等性总结

| 方法 | 幂等性 | 语义 | 示例 |
|------|--------|------|------|
| GET | ✅ 幂等 | 获取资源 | `GET /users` |
| HEAD | ✅ 幂等 | 获取响应头 | `HEAD /users` |
| OPTIONS | ✅ 幂等 | 获取支持的方法 | `OPTIONS /users` |
| PUT | ✅ 幂等 | 更新资源（全量） | `PUT /users/1` |
| DELETE | ✅ 幂等 | 删除资源 | `DELETE /users/1` |
| POST | ❌ 非幂等 | 创建资源 | `POST /users` |
| PATCH | ⚠️ 通常非幂等 | 部分更新 | `PATCH /users/1` |

**注意**：幂等性是 HTTP 语义的约定，不是强制约束。开发者应该遵循这些约定。

### 4. 异步方法如何同步调用

**问题**：一个异步方法，怎么能够同步调用？

**我的回答**：没有答上来 ❌

**补充要点**：

#### 场景分析

异步方法无法真正"变同步"，但有以下几种处理思路：

**1. async/await（推荐）**
```javascript
// 异步方法
async function fetchData() {
  const res = await fetch('/api/data')
  return res.json()
}

// 使用
async function main() {
  const data = await fetchData()  // 看起来同步，实际是非阻塞的
  console.log(data)
}
```

**2. Promise.then() 链式调用**
```javascript
fetchData().then(data => {
  console.log(data)
}).catch(err => {
  console.error(err)
})
```

**3. 回调函数（老式写法）**
```javascript
function fetchData(callback) {
  fetch('/api/data')
    .then(res => res.json())
    .then(data => callback(null, data))
    .catch(err => callback(err))
}

// 使用
fetchData((err, data) => {
  if (err) console.error(err)
  else console.log(data)
})
```

**4. 为什么不能真正同步？**

JavaScript 是单线程的，异步操作（如网络请求、定时器）由浏览器/Node.js 的事件循环处理。强行同步会阻塞整个线程：

```javascript
// ❌ 错误示例：这样会卡死浏览器
function syncFetch(url) {
  let result
  // 等待 Promise 的方式会阻塞主线程！
  fetch(url).then(res => result = res)
  while (!result) { /* 死循环等待 */ }  // 阻塞！
  return result
}
```

**正确的理解**：异步方法外部必须用 `async/await` 或 `.then()` 处理，这是 JavaScript 异步设计的核心，不是"污染"而是特性。
