## 二、STAR 法则话术结构

### 1. 情景 (Situation) 与痛点发掘

为了确保重构后的 AI 销售外呼系统真正契合业务，我前往贵阳一线职场进行了实地调研。在旁听座席外呼时，我发现了两处极具阻碍性的真实业务痛点：

- **痛点一：** 我们的核心用户群体普遍带有浓重的地方口音或直接使用方言，新座席往往难以听懂，导致意向客户流失。
- **痛点二：** 一线座席的物理机设备老旧且系统环境极度碎片化。

### 2. 任务 (Task) 与技术挑战

前端团队的任务随之明确：我们需要在毫秒级内解决座席的语言听力障碍，同时要在极其恶劣的浏览器沙箱环境中实现各业务系统间的连贯 Agent 动作调度。

### 3. 行动 (Action) 与架构决策

针对上述任务，我主导了以下核心技术落地与长远规划：

- **打通高兼容 ASR 链路：** 在技术方案里重点引入了具备强方言兼容能力的 ASR 语音流转写链路。系统在后台实时将方言转化为标准普通话文本，并结合 GLM 4.7 模型进行盘中实时话术推荐。

- **设计解耦 SDK：** 抛弃传统开发模式，设计并实现了与业务框架彻底解耦的 Web Components 和 Agent SDK。

- **推进云电脑架构演进：** 针对跨网页工具调用机制面临的兼容性痼疾，我正在主导推动业务端全面引入云电脑。这能从根本上拉平底层操作系统和浏览器环境，并确保所有通话和敏感数据在云端闭环，大幅提升数据合规安全性。

### 4. 结果 (Result) 与业务收益

该套方案上线后：

| 指标        | 收益                  |
| --------- | ------------------- |
| 新员工培训上岗周期 | 预计缩短 **30%**        |
| 单次通话意向转化率 | 实际提升约 **8%**        |
| 大模型能力接入成本 | 多条业务线接入成本降至 **1 天** |
