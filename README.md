# Vault README

这份 README 不是为了替代 `AGENTS.md`，而是为了在你日常使用时，快速判断一条内容该放哪里，以及什么时候应该升级。

详细规则看 `AGENTS.md`。这里只保留最常用、最容易混淆的部分。

## 这个 vault 在做什么

这是一个问题驱动的第二大脑。

目标不是多存内容，而是持续沉淀：

- 更好的问题
- 更稳定的理解
- 更清楚的判断
- 更可复用的表达

如果一条内容只是“看起来有意思”，但未来大概率不会反复复用，就默认不要进入正式结构。

## 顶层目录怎么理解

### `00-Capture`

临时收纳箱。

放这里的内容通常是：

- 刚看到、刚想到、刚摘下来的内容
- 还没判断价值和归属
- 单一案例、单篇文章、单次对话

一句话判断：

> 我现在只知道它可能有用，但还不知道它到底该怎么被我复用。

例子：

- “Lando Norris 官网做得很好，先记一下”
- “Shopify Editions 这个发布页的信息组织很强，先留个线索”
- “看到一篇文章，感觉有启发，但还没想清楚它和我现有知识有什么关系”

### `01-Projects`

有开始、有结束、有交付物、有截止或近期推进要求的事情。

一句话判断：

> 这件事需要我近期行动，而且做完就结束。

例子：

- “写一篇要发布的文章”
- “准备一次分享”
- “做一套个人网站改版方案，并在本月完成”

### `02-Areas`

长期责任领域。

比如健康、写作、投资、家庭、个人知识管理。

一句话判断：

> 这不是一次性任务，而是我长期要维护的责任。

例子：

- “写作能力建设”
- “个人知识管理系统维护”
- “健康管理”

### `03-Resources`

稳定参考知识库。

放这里的不是“我今天看到了什么”，而是“以后我还会反复回来查什么”。

一句话判断：

> 它已经从原始材料变成了稳定参考。

例子：

- “个人品牌网站如何建立记忆点”
- “高信息密度产品发布页设计原则”
- “UI 设计参考网站索引”

### `04-Archives`

已经完成、暂停、过时、不再维护的内容。

一句话判断：

> 保留，但不再让它占用当前注意力。

例子：

- “已经发布完的文章项目”
- “暂停维护的旧资源页”
- “已经完成的专题研究”

### `05-Daily`

按时间记录的粗糙输入、当天观察和未成熟想法。

一句话判断：

> 这条内容首先属于今天，而不是属于某个稳定主题。

例子：

- “今天突然想到一个问题”
- “今天读到一句话，先记下来”
- “今天做设计时遇到的卡点和临时判断”

### `06-Synthesis`

高纯度个人智慧区。

这里只放：

- 我的长期判断
- 我的结构化理解
- 我的方法论
- 我的跨领域整合

一句话判断：

> 这页最核心的价值，是“我怎么看”，不是“资料里写了什么”。

例子：

- “我如何判断一个个人品牌网站是否真正有记忆点”
- “我对 AI 参与设计工作流的长期判断”
- “我如何区分值得进入 vault 的知识和信息糖”

## 最容易混淆的三个目录

### `00-Capture` vs `03-Resources`

`00-Capture` 是原材料，`03-Resources` 是稳定参考。

不要因为一个案例、一个网站、一篇文章“看起来很好”就直接放进 `03-Resources`。

正确升级方式通常是：

1. 先放 `00-Capture`
2. 提炼出稳定主题
3. 再进入 `03-Resources`

例子：

- “Lando Norris 官网做得很好” -> 先放 `00-Capture`
- “个人品牌网站如何建立记忆点” -> 可以进入 `03-Resources`
- “我收集了 8 个官网案例” -> 先放 `00-Capture`
- “个人品牌网站常见的 4 种记忆点来源：配色、动效、叙事、人格化细节” -> 可以进入 `03-Resources`

### `03-Resources` vs `06-Synthesis`

`03-Resources` 解决的是“这类东西一般该怎么理解”。

`06-Synthesis` 解决的是“这对我意味着什么，我以后怎么判断类似问题”。

例子：

- “高信息密度产品发布页设计原则” -> `03-Resources`
- “我如何权衡信息密度、视觉冲击和转化路径” -> `06-Synthesis`
- “个人品牌网站常见设计模式” -> `03-Resources`
- “我不再追求炫技式个人网站，而是优先判断它是否服务于身份、叙事和转化” -> `06-Synthesis`

### `00-Capture` vs “信息糖”

不是所有内容都值得进入 `00-Capture`。

如果它只是：

- 热点刺激
- 低验证观点
- 短期情绪满足
- 未来几乎不会复用的碎片

那它可能连 `00-Capture` 都不值得长期保留。

例子：

- “这个网站好酷” -> 太空，不够
- “这个热点观点让我很兴奋” -> 先问未来是否会复用
- “这篇内容虽然刺激，但没有带来稳定问题或判断” -> 通常不值得写入

## 一个内容进来时，先问这 6 个问题

1. 它现在需要我行动吗？
2. 它是长期责任的一部分吗？
3. 它未来会被我反复检索吗？
4. 它只是原材料，还是已经形成稳定理解？
5. 它真正改变了我什么判断？
6. 它会降低理解成本，还是增加维护成本？

## 从案例到知识的升级路径

很多内容一开始只是案例，不应该直接进入正式 wiki。

推荐路径：

1. 我看到了一个好案例
2. 我记录它为什么值得注意
3. 我提炼出它回答的稳定问题
4. 我把稳定问题整理成 `03-Resources`
5. 我在长期复用后形成自己的判断
6. 只有这时，才考虑进入 `06-Synthesis`

例子：

1. 我看到 `Lando Norris` 官网，觉得做得很好
2. 我先记下它为什么让我印象深刻
3. 我发现它真正回答的问题是“个人品牌网站如何建立记忆点”
4. 我把这个问题整理成 `03-Resources` 主题页
5. 我以后反复用这套框架判断别的网站
6. 我开始形成“我自己如何判断个人品牌网站”的标准，这时才接近 `06-Synthesis`

## 什么时候可以进 `03-Resources`

满足越多，越适合进：

- 标题已经是稳定概念，不是单一事件或单一案例名
- 去掉原始案例后，页面依然成立
- 内容回答的是“方法、模式、边界、结构”
- 未来三个月到一年后仍值得检索
- 它能和现有资源页形成链接，而不是制造分叉

快速判断：

> 如果删掉案例名，这页还能成立，它就可能适合 `03-Resources`。

例子：

- “Lando Norris 官网案例观察” -> 还不适合
- “个人品牌网站设计：记忆点、叙事与动效边界” -> 更适合
- “Shopify Editions Winter 2026 分析” -> 还不适合
- “高信息密度产品发布页设计原则” -> 更适合

## 什么时候可以进 `06-Synthesis`

满足越多，越适合进：

- 页面核心价值来自我的判断
- 我能明确说出“这对我意味着什么”
- 我以后会用这页指导选择、行动或表达
- 没有“我的思考”，这页就失去核心价值
- 它已经不是资料整理，而是我的认知结论

快速判断：

> 如果删掉“我的思考”，这页就不成立，它才接近 `06-Synthesis`。

例子：

- “个人品牌网站设计模式整理” -> 更像 `03-Resources`
- “我为什么认为多数个人品牌网站把技术炫技误当成品牌表达” -> 接近 `06-Synthesis`
- “高信息密度页面的信息架构方法” -> 更像 `03-Resources`
- “我如何判断一个页面该追求信息密度还是叙事体验” -> 接近 `06-Synthesis`

## 默认处理原则

- 不确定时，先放 `00-Capture`
- 想长期复用时，提炼后进 `03-Resources`
- 形成个人判断后，再考虑 `06-Synthesis`
- 如果复用价值低于维护成本，默认不写入

## 页面最小规范

所有 Markdown 笔记都应有 frontmatter，最少包含：

```yaml
---
tags:
  - 示例标签
type: resource
status: active
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

## 最后一个提醒

不要把“收集到了很多好东西”误当成进展。

真正的进展通常是这三件事之一：

- 我提出了更好的问题
- 我把一个模糊感觉提炼成了稳定理解
- 我形成了自己的判断
