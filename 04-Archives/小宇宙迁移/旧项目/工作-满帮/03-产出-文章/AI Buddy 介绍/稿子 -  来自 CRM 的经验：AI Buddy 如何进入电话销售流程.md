## CRM 团队的实践路径：让 AI Buddy 真正“进入”电话销售

-- AI Buddy 在里面起了什么作用

在 CRM 团队的探索中，我们对 AI Buddy 的认知越来越清晰：它不是要取代销售人员，而是成为销售人员的“实时智能副驾”。AI Buddy 主要负责处理信息检索、数据分析、工具调用和内容生成，让销售人员能够专注于建立信任、挖掘需求和推动成交。

一个很典型的工作流是这样的：

通话前，销售人员先在 CRM 里查看 AI Buddy 自动生成的客户画像总结，快速进入状态，明确这通电话的目标。

进入沟通后，销售人员围绕客户当前需求自然推进对话。当用户提出需要进一步处理、需要补充信息或者要现场完成填单、领取优惠、确认订单等情况时，销售人员不再需要一边沟通一边手忙脚乱地切换页面、整理字段，而是可以在合适的时机点击“AI 介入”按钮，让 Agent 介入分析当前对话内容，提取关键信息，并调用工具去完成后续动作（生成个性化方案、创建订单、推送动态卡片等）。

相比传统 CRM 模式，这种方式带来了显著提升：

- **传统 CRM**：销售人员需要频繁切屏查询、手动填写信息，容易打断对话节奏，增加出错风险，也影响客户体验。
- **AI Buddy**：AI 在后台默默支撑，销售人员保持流畅、自然的沟通，客户感受到的是专业、快速、个性化的服务。

为了支撑这种深度嵌入的协同模式，我们将 AI Buddy 的能力拆分成以下几个核心、可复用的模块：

- **Agent 注册机制**：业务团队可以轻松将自己的业务逻辑（如优惠规则、库存查询、订单生成等）注册成后台 Agent，一次注册，多场景复用。
- **客户端工具调用**：AI Buddy 可无缝接入公司已有的地图、支付、CRM 系统、物流等接口，实时拉取数据并执行操作。
- **Generative UI（生成式界面）**：根据对话上下文动态生成卡片、表单、推荐列表或报价单，无需前端硬编码，实现“所见即所得”。
- **Chat UI 集成套件**：支持快速集成到小程序、App、H5 等多种前端框架，几行代码即可实现上下文记忆、多轮对话和工具调用功能。

通过这些模块的有机组合，AI Buddy 真正融入了电话销售的完整流程。

### AI Buddy SDK：三步完成接入，让应用"开口说话"

AI Buddy SDK 的核心理念是：**让业务团队像调接口一样调用 AI 能力**，无需关心大模型、Prompt 工程、上下文管理等复杂技术细节。

接入过程非常简单，只需三步，[快速开始](https://qa-ai-buddy.amh-group.com/get-started)：

**第一步：引入并初始化**
```javascript
import { AiBuddy } from '@ai-buddy/sdk'

const agent = new AiBuddy({
  agentId: 'demo-agent',
  name: 'Demo Assistant',
  agent: {
    api: 'https://your-agent-api.example.com/chat',
    instructions: 'You are a concise assistant for the host app.',
    credentials: 'include',
  },
})
```
配置 `agentId` 标识业务能力，`instructions` 会追加到系统提示词后面，用来补充当前应用的约束与语气。

**第二步：挂载 UI**
```javascript
// 浮动模式（推荐快速验证）
agent.mount({
  mountConfig: {
    ui: {
      title: 'Demo Assistant',
      layout: { mode: 'floating' },
    },
  },
})

// 容器模式（需要提供容器元素）
agent.mount({
  container: '#chat-root',
  mountConfig: {
    ui: { layout: { mode: 'container' } },
  },
})
```
支持浮动和容器两种布局模式，一行代码即可在小程序、App、H5 等任意前端框架中展示对话界面。

**第三步：发送消息**
```javascript
agent.sendMessage('Hello')
agent.sendMessage('Draft only', { autoSend: false })  // 草稿模式
```
就这样，你的应用已经具备了智能对话能力。SDK 会自动处理对话上下文、调用后台工具、返回结构化结果。

## 落地效果：真实电话场景里的转化提升
-- 范式转移

在原来的范式里，销售既要负责和客户沟通，也要同时承担大量伴随式的信息处理工作: 查资料、记重点、补字段、填小结单、安排后续动作。这些事情每一项都合理，但当它们不断叠加在同一个人身上时，真正被挤压掉的，往往是对客户意图的理解、对沟通节奏的判断，以及对关键机会的把握。

而在新的范式里，AI Buddy 和 Agent 开始接住那些可以标准化、可以被工具化的部分。销售的角色并没有被削弱，反而变得更聚焦了: 把注意力放在理解用户、推进沟通、判断下一步动作上；把原本机械、重复、容易打断节奏的动作，交给 Agent 去辅助完成。销售不再需要把大量时间浪费在填写单子上，而是更多地审核 Agent 的结果、确认信息是否准确，并把精力放回真正高价值的沟通环节。

这种变化的意义，不只是节省了几个操作步骤，而是让一线业务里最稀缺的东西重新被释放出来了。对销售来说，这是心智负担的下降；对用户来说，这是更少的等待和摩擦；对团队来说，这是更稳定的执行过程，以及更有机会被复制的方法。
