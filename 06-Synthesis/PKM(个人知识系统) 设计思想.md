# PKM(个人知识系统) 设计思想

## 什么是 PKM

PKM 不是信息囤积系统，也不是资料收藏夹。

它的目标不是“拥有更多信息”，而是持续构建一个问题驱动的第二大脑：帮助我形成更强的判断、更清晰的结构、更独特的认知差，并最终转化为行动与表达。

信息本身不是价值。只有当信息被组织、提炼、连接，并进入我的思考与复习回路时，它才有可能变成认知资产。

## 为什么传统 PKM 容易失效

传统 PKM 很容易滑向“收藏癖”：

- 捕获越来越容易
- 存储成本越来越低
- 检索越来越强
- 但判断、提问、整合并没有同步增强

结果是，信息越积越多，理解没有同步加深，系统逐渐变成一个看似有序、实际失控的信息仓库。

在 AI 时代，这个问题会被进一步放大。因为 LLM 可以极大提升捕获、总结、改写和连接的效率，也因此更容易制造“我已经处理过”的错觉。

所以 PKM 必须明确反对一个方向：为了记录而记录，为了完整而堆积，为了安全感而收藏。

## PKM 的真正目标

这个系统真正服务的，不是资料管理，而是认知演化。

它应该帮助我：

- 识别信息差
- 提出更好的问题
- 发现矛盾、缺口与异常
- 形成跨领域连接
- 提炼成稳定的思考框架
- 转化为行动、表达与复利

最终，PKM 的价值不体现在“存了多少”，而体现在：

- 我是否更容易提出关键问题
- 我是否更快看见结构与矛盾
- 我是否形成了不可复制的判断方式
- 我是否能持续输出有力量的表达

## AI 时代的人机分工

AI 应该负责 bookkeeping，人类负责判断。

LLM 擅长：

- 捕获
- 总结
- 归类
- 去重
- 关联
- 检查一致性
- 发现冲突
- 维护结构

人类必须保留：

- 提问
- 选择
- 判断
- 赋予意义
- 确定优先级
- 形成观点
- 做出行动承诺

LLM 可以拥有大量世界知识，但它不会自然承担“什么问题真正重要”这件事。真正的护城河不再是信息获取，而是问题质量、判断能力和持续提炼的能力。

## 为什么以问题为驱动

答案会快速过时，问题会持续推动系统进化。

一个健康的 PKM，不应该只积累“我知道了什么”，还应该持续暴露：

- 我还没想清楚什么
- 哪些地方存在冲突
- 哪些结论只是暂时成立
- 哪些问题值得长期跟踪
- 哪些线索可能改变我对某个领域的理解

问题不是答案的附属品，而是认知系统的发动机。

如果一个系统只能储存答案，却不能持续生成、保留、回访高质量问题，那么它最终只会变成静态档案，而不是动态认知引擎。

## 为什么采用 CODE + PARA

CODE 解决的是信息流动过程：

- Capture：先捕获，降低进入阻力
- Organize：按行动关系组织，而不是按主题堆放
- Distill：提炼重点、结构与判断边界
- Express：把知识转化为输出，而不是停留在库存

PARA 解决的是知识放置逻辑：

- Projects：短期要推进的事项
- Areas：长期要维护的责任
- Resources：供未来调用的参考知识
- Archives：已完成、暂停、过时的内容

这两个框架结合后，PKM 才不会停在“收集”，而能够支持从输入到输出的完整循环。

## 第二大脑的本质

第二大脑不是替我思考，而是支持我更好地思考。

它的作用不是替代记忆，而是：

- 减轻机械记忆负担
- 保留关键上下文
- 让重要问题反复出现
- 帮助我跨时间回看自己的思路
- 防止有价值的线索在遗忘中流失

真正重要的不是“记住更多”，而是“让真正重要的东西被重新看见”。

## 遗忘与主动回忆

人会遗忘，认知也会退化。

所以 PKM 不能只是静态存储系统，它还必须承担“帮助回忆”的作用。一个没有回访机制的知识库，最终只会沉没为档案库。

这意味着，一个有效的 PKM 不仅要支持记录，还要支持：

- 重新浮现重要问题
- 暴露长期未处理的开放循环
- 提醒值得继续验证的判断
- 帮助我看见哪些内容进入了遗忘区
- 把过去的输入重新带回当前思考

知识只有在被重新调用、重新审视、重新表达时，才真正进入认知。

## 反熵原则

所有知识系统都会熵增，PKM 也不例外。

熵增的表现包括：

- 重复笔记不断增加
- 同一概念出现多个版本
- 捕获区长期堆积
- 页面彼此孤立
- 只有摘录，没有判断
- 页面越来越多，但理解没有变深

因此，PKM 的核心任务之一不是扩容，而是垃圾回收。

所谓维护，不是制造更多页面，而是持续压缩混乱、消除分叉、减少噪音、提升结构自洽性。

## PKM 最终服务什么

这个系统最终服务的不是存档，而是表达。

表达包括：

- 写作
- 决策
- 对话
- 研究
- 分享
- 行动
- 长期判断的形成

如果知识没有进入表达，它就很难真正完成转化。

所以 PKM 的终点不是“我存下来了”，而是：

- 我提出了什么更好的问题
- 我能输出什么有力量的东西
