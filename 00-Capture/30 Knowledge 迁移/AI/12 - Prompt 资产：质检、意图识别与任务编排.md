# Prompt 资产：质检、意图识别与任务编排

好 Prompt 的价值不在于华丽措辞，而在于把角色、上下文、目标、约束和输出协议压成一个稳定接口。

## Prompt 的四种常见用途

- 评审型：让模型扮演挑剔的专家，对你的解释或方案做追问。
- 编排型：给模型明确工程目标、约束和执行顺序。
- 分类型：把意图识别、质检、审核等任务压成规则化判断。
- 抽取型：把非结构化对话转成稳定 JSON 输出。

## 稳定 Prompt 的组成

- Role：模型扮演谁。
- Context：当前项目、业务背景和前置限制。
- Objective：任务到底要完成什么。
- Constraints：哪些不能做，哪些必须遵守。
- Output：输出格式必须多严格。

## 质检和意图识别的关键点

- 规则要写成“命中条件”，不要只写主观描述。
- 要明确只看哪部分输入，避免上下文污染。
- 要给否定处理规则，不然容易误命中。
- 输出格式要锁死，尤其是需要 JSON 时。

## 用 Prompt 做面试训练

这是一个高杠杆用法：

- 让模型模拟挑剔面试官。
- 限定问题范围，比如流式输出、跨页状态同步、安全。
- 要求模型在你回答后继续追问，而不是一次性给标准答案。

## 用 Prompt 做工程编排

当任务跨 Page、Content Script、Background、MCP SDK 等多个层级时，Prompt 里必须提前声明：

- 用什么库。
- 哪些抽象优先复用。
- 哪些底层实现禁止重写。
- 代码必须拆成哪些模块。

一句话总结：
Prompt 不是给模型“说清楚”，而是把任务变成一个可重复执行、可验证、可审计的接口定义。
