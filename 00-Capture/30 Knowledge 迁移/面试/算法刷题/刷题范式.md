# 刷题范式

> 合理运用心流通道，科学刷题，快乐刷题！

---

## 方法 A：专题训练（推荐新手）

### 核心思路

1. **按照专题刷题**，而不是随机刷题
   - 同一个专题，一个套路可以解决多个题目，刷题效率高
   - 从不同的角度去观察、思考同一个算法，从而深刻地理解算法的本质

2. **螺旋上升式学习**
   - 把各个知识点的基础题刷一遍，再刷更难的题目
   - 难度分低的题目一般只会考察一个知识点
   - 难度分高的题目会同时考察多个知识点

### 推荐专题顺序

| 顺序 | 专题 | 备注 |
|-----|------|------|
| 1 | 滑动窗口 | 定长/不定长 |
| 2 | 二分算法 | 二分查找 |
| 3 | 数据结构 | 前缀和、栈、队列、堆 |
| 4 | 二叉树 DFS | 理解递归 |
| 5 | 网格图 DFS | 进一步理解递归 |
| 6 | 回溯 | 进一步理解递归 |
| 7 | 动态规划 | 唯有坚持多练，至少要做 50 道才算入门 |

## 方法 B：随机训练（适合进阶）

### 适用场景

渡过萌新阶段后，可以偶尔随机刷题，锻炼自己判断题目类型的能力。

### 操作步骤

1. 打开 [难度练习](https://huxulm.github.io/lc-rating/v0/zen)
2. 在设置中关闭算法标签
3. 选择适合自己的难度范围

### 优缺点

| 优点 | 缺点 |
|-----|-----|
| 做题时不知道题目类型，可以增强实战能力 | 知识点零散，不如题单那么系统 |
| 查漏补缺，检验自己的学习成果 | |

## 方法 C：突击训练（适合面试）

### 适用场景

如果即将面试，时间紧迫，可以刷经典面试题。

### 推荐内容

1. **HOT 100** - 经典面试题（首选）
2. **面试 150** - 与 HOT 100 有很多重复，有时间可补充

## 常见问题

### Q: 做题没思路，思考多久可以看题解？

**A**:
- 十分钟到数小时都可以
- 如果看完题解觉得题解很妙，那就学到了一个自己不会的技巧
- 可以换个时间段思考 —— **孵化效应**：离开问题后，大脑会在无意识中处理问题

### Q: 如何根据数据范围估计时间复杂度？

**A**: 一般每秒能执行约 10^8 次运算

| 数据范围 | 允许的时间复杂度 | 适用算法举例 |
|---------|-----------------|-------------|
| ≤ 12 | O(n!) 或 O(2^n × n) | 回溯、暴力搜索 |
| ≤ 18 | O(2^n) | 状态压缩 DP |
| ≤ 30 | O(2^{n/2}) | 折半枚举 |
| ≤ 100 | O(n^3) | 三重循环 |
| ≤ 500 | O(n^2) | 二重循环 DP、背包 |
| ≤ 10^5 | O(n log n) 或 O(n) | 线性 DP、滑动窗口 |
| ≥ 10^9 | O(1) 或 O(log n) | 二分、快速幂、数学公式 |

> **注意**：实际做题时，注意常数因子的影响。例如哈希表比数组慢。
