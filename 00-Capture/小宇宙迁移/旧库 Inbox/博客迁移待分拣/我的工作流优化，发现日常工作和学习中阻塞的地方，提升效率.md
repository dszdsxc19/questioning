最终需要形成肌肉记忆，这样就不会给大脑带来负担。否则每次思考工作流反而增加认知负担，在自己的认知负担范围内优化工作流，不要强求强大的工作流，认识到精力是有限的，如果花费过多精力在工作流优化上，反而得不偿失。系统地看待，是一个整体，而且有反馈延迟。

核心原则：手不离键盘

0. 全局极速搜索 (Raycast) - 不在 Chrome 时用这个
   • 核心逻辑: Search then Switch (先搜再跳，而不是先切应用)。

• 配置: 在 Raycast 设置 -> Extensions -> Quicklinks 中设置别名 (Alias)。

• 肌肉记忆:

• Google: `Alt/Cmd+Space` -> 输入 `g` + 空格 + 关键词 -> 回车。

• GitHub: `Alt/Cmd+Space` -> 输入 `gh` + 空格 + 库名 -> 回车。

• 计算/汇率: 直接输入 "100 usd in cny" 或 "12\*99"。

1. 窗口切换 (AltTab) - 多窗口管理核心
   • 为什么必须取代 macOS 原生切换?

• 原生痛点: macOS 的 `Cmd+Tab` 是基于应用 (App-based) 的，无法区分同一应用的多个窗口。更致命的是，在多桌面 (Spaces) 环境下，原生切换经常导致「只切换了顶栏菜单，窗口却没聚焦」或者「不知道切到了哪个桌面」的迷失感。

• AltTab 优势: 它是基于窗口 (Window-based) 的逻辑（类似 Windows），能列出所有桌面的所有窗口预览图，并强制跨桌面精准聚焦，彻底解决焦点丢失问题。

• 配置: 必须在设置中勾选 "Show windows from All Spaces"。

• 操作: `Cmd + Tab` -> 看到预览图 -> 松手。

2. Chrome 内搜索 (Cmd+L) - 已在 Chrome 时用这个
   • 场景: 刚切回 Chrome 或就在当前页面。

• 操作: `Cmd + L` (强制聚焦地址栏) -> 输入 -> 回车。

• 优势: 无视任何网页输入框的阻挡 (Vimium 的盲区)，绝对可靠。

3. Chrome 页面与标签 (Vimium) - 导航核心
   • 新建与打开 (高频):

• `o` (Vomnibar): 在当前标签页打开。可以输入历史记录、书签或直接搜索（功能类似 Chrome 内部的 Raycast）。

• `t` (New Tab): 新建标签页。

• 搜标签 (神技): `Shift + T` (输入关键词跳转已开标签)。

• 页面内搜框: `gi` (跳到第一个输入框) / `f` (显示所有点击快捷键)。

• 左右切: `J` (左) / `K` (右)。

• 历史回跳: `^` (Shift + 6) -> 在 A/B 两个页面反复横跳。

4. 异常处理
   • Vimium 卡住?: 按 `Esc` 退出网页输入框模式。
